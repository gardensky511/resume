##### 大規模フロントエンドにおける状態管理のリファクタリング

- 課題・背景 
  - Redux の store がページ単位で生成され定義が不統一なため、状態不整合による障害が頻発
  - 共通処理における TypeScript の型定義が不正確（`any` の多用）で、200箇所以上の潜在的な実行時エラーが存在
  - メンテナンス停止ライブラリ（`next-redux-wrapper`）への依存により、Next.js のバージョンアップが不可能な状態
- 行動・成果
  - 安全性を重視した段階的リファクタリング計画を立案し、影響範囲 41 ページに及ぶ大規模改修を障害なくリリース
  - 従来の手動テストではテストに数ヶ月を要する状況だったため、単体テストで安全性を担保できる体制を提案・構築し、テスト工数を大幅削減
  - 全ページの store 定義を統一し、store 不整合起因の障害を根本的に解消
  - 200箇所以上の潜在的実行時エラーを解消し、アプリケーションの堅牢性を大幅に向上
  - `next-redux-wrapper` の機能を自作処理で代替し、Next.js のバージョンアップが可能な状態を実現

##### CI 導入と品質ゲート整備
- 課題・背景
  - CI が未整備で、型エラーや単体テスト失敗のままでもマージされる事例が発生
